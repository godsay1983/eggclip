# EggClip HarmonyOS 上架审核整改方案

## 1. 文档目的

本文针对 `docs/20260713上架审核问题/` 中的审核报告、自检报告、截图和日志，定义 EggClip HarmonyOS 版重新提审前的整改方案。

本次整改只解决上架合规、审核可验证性和相关界面体验问题，不修改 EggClip 的局域网产品定位、配对安全模型或 HarmonyOS 剪贴板授权边界。

配套执行计划见 [`20260713鸿蒙上架审核整改ROADMAP.md`](./20260713鸿蒙上架审核整改ROADMAP.md)。

## 2. 证据范围

本方案分析了以下材料：

- `上架审核报告/问题.txt`：正式审核反馈。
- `上架审核报告/1.jpeg` 至 `4.jpeg`：正式审核问题截图。
- `上架审核报告/日志/`：4 台测试设备的色彩检测日志。
- `自检/自检.txt`：AppGallery 上架自检反馈。
- `自检/1.jpeg` 至 `12.jpeg`：手机、折叠屏和平板的自检过程截图。

四份检测日志都指向低对比度控件，不代表四类独立缺陷。正式审核和自检结果可归纳为四个上架阻断项，以及两个需要一并修正的界面问题。

## 3. 整改目标

重新提审前应达到以下结果：

1. 所有小号文字的对比度不低于 `4.5:1`，图标、控件边界和大号文字不低于 `3:1`。
2. AppGallery 上传图标不含任何透明像素。
3. 审核员不搭建 Windows 环境，也能通过审核说明和视频完整判断核心功能。
4. AppGallery 隐私标签、应用内隐私说明和对外隐私政策保持一致。
5. 普通用户界面不再直接展示 POC、WebSocket、mDNS、`running`、`idle` 等开发术语。
6. 手机和平板的底部导航不遮挡页面内容。
7. 整改不引入永久邀请、认证后门、模拟连接或伪造业务数据。

## 4. 审核问题与根因

| 优先级 | 问题 | 审核表现 | 根因 |
| --- | --- | --- | --- |
| P0 | 色彩对比度不足 | 浅色模式最低 `1.64:1`，深色模式最低 `1.59:1` | 品牌黄色被直接用作浅色表面上的文字和图标颜色 |
| P0 | 上架图标透明 | AppGallery 提示图标存在透明背景 | 上传了透明导出图，而不是专用的不透明市场图标 |
| P0 | 核心功能无法审核 | 审核员没有可信桌面端，无法扫码或粘贴邀请 | 缺少完整演示视频、配套客户端说明和审核步骤 |
| P0 | 隐私标签缺失 | 检测到个人信息处理行为，但控制台未勾选 | 把“无云端收集”误认为“不处理个人信息” |
| P1 | 诊断信息过多 | 设备页出现 POC、端点、运行状态和协议术语 | 调试界面没有与普通用户界面完全分层 |
| P1 | 浮动导航遮挡内容 | 手机和平板的底部内容被导航覆盖 | 页面没有统一预留导航高度和系统安全区 |

## 5. 色彩系统整改

### 5.1 根因说明

当前浅色主题主色为 `#F4C430`。它与白色背景的对比度约为 `1.64:1`，与审核结果一致。主色适合用作较大面积的按钮背景或装饰，不适合用作白色、米白色或浅黄色表面上的小号文字。

当前高风险位置包括：

- 底部导航选中项文字和图标。
- “连接中”等等待状态文字。
- 使用 `EggClipColors.primary` 作为 `fontColor`、图标描边或小面积前景色的其他控件。

### 5.2 颜色语义

保留品牌黄色，但拆分颜色职责：

| Token | 用途 | 浅色主题建议 | 深色主题建议 |
| --- | --- | --- | --- |
| `primary` | 主按钮背景、选中底板、装饰强调 | 保留现有蛋黄色 | 保留现有深色主题黄色 |
| `onPrimary` | 黄色底板上的文字和图标 | 深棕色 | 深棕色 |
| `accentForeground` | 浅色表面上的强调文字 | `#6B4700` 或等效高对比颜色 | `#FFD75A` 或等效高对比颜色 |
| `statusPending` | 连接中、等待确认 | 使用可读的琥珀色前景和低饱和底色 | 使用可读的亮琥珀色前景和深色底色 |
| `textPrimary` | 导航文字、标题、正文 | 现有主文字色 | 现有主文字色 |

最终颜色可以在视觉联调时微调，但必须先通过程序化对比度检查，再进入真机截图复核。

### 5.3 组件规则

- 底部导航选中态使用“黄色底板 + `onPrimary` 图标”，文字保持 `textPrimary`；不能继续使用白底黄字。
- “连接中”使用独立的 `statusPending`，不能直接使用 `primary`。
- 禁用态仍需能被识别；若文字承载信息，不以极低透明度代替禁用表达。
- 同一种状态在首页、设备页、配对页和设置页必须使用相同 token。
- 全局检查 `fontColor(EggClipColors.primary)`、图标 `color: primary` 和描边色，不能只修审核截图中的控件。

### 5.4 验收方法

1. 为主题 token 建立颜色配对清单并计算 WCAG 对比度。
2. 覆盖浅色、深色及系统跟随三种主题。
3. 覆盖首页、设备、设置、配对和诊断页面。
4. 覆盖正常、选中、连接中、在线、离线、认证失败、禁用等状态。
5. 在手机、折叠屏和平板真机截屏复核。
6. 重新运行 AppGallery 上架自检，不再出现色彩告警。

## 6. 上架图标整改

### 6.1 资源职责

HarmonyOS 运行时继续使用分层图标：背景层保持不透明，前景层允许透明。AppGallery 市场图标应使用单独的扁平化导出文件。

不要为了修复市场图标而取消运行时前景层透明，否则会破坏 HarmonyOS 分层图标效果。

### 6.2 市场图标要求

- 新增独立的市场图标资源，例如 `docs/store-assets/app-icon-opaque.png`。
- 使用与应用视觉一致的纯色背景，建议从 `#FFF8E7` 开始验证。
- 所有像素 Alpha 必须为 255。
- 图案四周保留安全边距，不贴边、不裁切。
- 按 AppGallery Connect 当前要求导出尺寸；不要直接上传文档中的透明展示图。
- 上传前自动检查尺寸、色彩模式和透明通道。

### 6.3 验收标准

- 自动检查结果显示透明像素数量为 0。
- 在浅色和深色市场页面预览中边界清楚。
- HAP 中的分层图标仍正常显示。
- AppGallery Connect 中重新上传并保存新的市场图标。

## 7. 审核可验证性整改

### 7.1 原则

EggClip 依赖 Windows 配套客户端、同一局域网和一次性邀请。审核员无法自行构造可信环境时，应通过完整演示材料说明功能，不能通过降低安全性解决。

禁止以下做法：

- 内置永久有效的邀请字符串。
- 延长邀请到不合理的有效期或取消一次性消费。
- 添加绕过认证的审核入口。
- 伪造在线设备、同步结果或历史记录。

### 7.2 完整演示视频

视频应连续展示以下流程：

1. Windows 桌面端启动、版本号和局域网状态。
2. Windows 与 HarmonyOS 设备位于同一局域网。
3. 桌面端生成一次性邀请二维码。
4. HarmonyOS 端扫码或粘贴邀请。
5. 双端核对确认码并完成认证。
6. 可信设备在两端显示在线。
7. Windows 复制文本，HarmonyOS 实时收到并显示。
8. HarmonyOS 通过系统 `PasteButton` 授权发送文本，Windows 系统剪贴板自动更新。
9. HarmonyOS 应用退出并重新启动后，可信设备自动重连。
10. 展示最近记录、历史开关、移除设备等基础能力。
11. 简要展示手机和平板布局。

视频中不得展示可复用的邀请秘密、密钥、完整安全材料或真实私人剪贴板内容。演示文本使用无敏感信息的固定样例。

### 7.3 审核备注

提审备注至少包含：

- 应用不需要账号。
- 需要 Windows 配套客户端和同一局域网。
- Windows 客户端版本和获取方式。
- 演示视频链接。
- 从生成邀请到完成双向同步的编号步骤。
- HarmonyOS 发送剪贴板必须点击系统 `PasteButton`。
- 应用不使用中心服务器、公网中继、广告或统计 SDK。
- 隐私政策链接和联系渠道。

### 7.4 应用内帮助

在未配对状态提供“使用前准备”入口，包含：

1. 安装并启动 Windows 客户端。
2. 两端连接同一 Wi-Fi 或可互通局域网。
3. 在桌面端生成邀请。
4. 使用扫码或粘贴完成配对。
5. 无法连接时进入连接诊断。
6. 查看功能演示和隐私说明。

帮助页面是静态教程，不模拟连接，也不展示伪造数据。

## 8. 隐私标签整改

### 8.1 数据清单

以 AppGallery Connect 当前提供的分类名称为准，至少核对以下数据：

| 数据 | 处理目的 | 处理范围 | 建议声明口径 |
| --- | --- | --- | --- |
| 剪贴板文本、用户内容 | 核心剪贴板同步 | 用户授权后经局域网传给已配对设备 | 用户主动提供；用于应用功能 |
| 应用生成的设备 ID、公钥、可信状态 | 配对和身份认证 | 本地保存，配对时交换必要公开信息 | 用于应用功能和安全认证 |
| IP 地址、端口、局域网发现信息 | 发现并连接可信设备 | 本机局域网内处理 | 用于网络连接和应用功能 |
| 本地历史记录 | 展示最近剪贴板记录 | 本机保存，受历史策略控制 | 用于应用功能；用户可关闭和清理 |
| 加密密钥和 HUKS 引用 | 加密与认证 | 设备安全存储内处理 | 不作为普通业务数据上传或共享 |

### 8.2 统一口径

以下三处必须一致：

1. AppGallery Connect 隐私标签。
2. 对外可访问的隐私政策页面。
3. 应用内“隐私与安全”说明。

必须明确说明：

- 不需要账号。
- 不连接开发者中心服务器。
- 不使用公网中继。
- 不集成广告、统计或崩溃上报 SDK。
- 剪贴板内容只在用户授权后，通过局域网传输给已配对设备。
- 历史记录保存在本机，用户可以关闭或清理。
- 普通日志不记录剪贴板正文、邀请秘密和密钥。

“不上传到开发者服务器”不能替代隐私标签填写，因为应用仍然处理并向用户指定的已配对设备传输内容。

## 9. 普通界面精简

### 9.1 设备页

普通状态只展示：

- 已连接。
- 连接中。
- 未连接。
- 认证失败。
- 已暂停。

以下信息移入折叠的“连接诊断”：

- mDNS 状态。
- WebSocket 状态。
- IP 和端口。
- 帧统计。
- 最近失败阶段。

普通界面不显示 `POC`、`running`、`idle`、内部消息名或 `127.0.0.1:0` 占位端点。没有真实运行时连接时，不渲染运行时设备卡片。

### 9.2 底部导航

- 页面正文底部预留“导航栏高度 + 系统安全区 + 额外间距”。
- 手机、折叠屏和平板使用统一的安全区算法。
- 浮动导航固定，正文独立滚动。
- 最后一张卡片和最后一个操作必须能完全滚动到导航栏上方。

### 9.3 发现操作

“开始发现”和“停止发现”合并为一个状态按钮：

- 未运行：开始发现。
- 运行中：停止发现。
- 已发现：展示设备并提供连接操作。

## 10. 回归与提审门禁

### 10.1 自动检查

- Harmony 单元测试通过。
- Harmony HAP 构建通过。
- 市场图标透明像素检查通过。
- 颜色 token 对比度检查通过。
- 发布安全检查不输出或包含敏感信息。

### 10.2 真机回归

至少覆盖：

- 一台 HarmonyOS 手机。
- 一台 HarmonyOS 平板。
- 浅色、深色和系统跟随主题。
- 未配对、连接中、在线、离线、认证失败和自动重连状态。
- Windows 到 HarmonyOS 的实时同步。
- HarmonyOS `PasteButton` 到 Windows 的实时同步。
- 应用退出后重新启动并自动重连。
- 历史记录开关和清理。
- 底部导航与最后一项内容不重叠。
- 隐私说明在未配对状态可访问。

### 10.3 提审门禁

以下项目全部满足后才能重新提交：

- AppGallery 自检不再报告色彩对比度问题。
- 市场图标已替换为不透明版本。
- 隐私标签已填写并复核。
- 隐私政策已发布且应用内可访问。
- 演示视频已上传并可在无登录状态观看。
- 审核备注包含完整操作步骤。
- 正式签名 HAP 完成手机和平板回归。
- 版本号和版本码已按发布规则递增。

## 11. 非目标

- 不增加账号、云同步、遥测、广告、自动更新或公网中继。
- 不修改 Ed25519、X25519、HKDF、AES-GCM 等既有安全方案。
- 不允许 HarmonyOS 静默读取系统剪贴板。
- 不改变邀请 5 分钟过期和一次性消费规则。
- 不以审核为由添加认证后门、永久测试设备或假数据模式。
- 不扩展到图片、文件或富文本同步。
