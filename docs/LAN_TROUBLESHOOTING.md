# EggClip 局域网连接排障

## 快速检查

1. 确认桌面端正在运行，HarmonyOS 应用位于前台。
2. 确认两端连接同一个可信局域网，路由器没有开启 AP/客户端隔离。
3. 查看桌面端诊断中的 IPv4 和 WebSocket 端口，优先选择物理 Wi-Fi/以太网地址。
4. 允许 EggClip 通过 Windows 专用网络防火墙；不要直接关闭整个防火墙。
5. VPN、TUN、虚拟网卡或模拟器桥接可能改变路由。发现地址不正确时，在 HarmonyOS 配对页选择正确 IPv4。

## 能发现但不能连接

- mDNS 只负责地址发现，不代表设备可信。确认邀请、六位确认码和同步空间一致。
- 检查桌面端显示的端口是否仍有效；桌面端重启后应优先使用邀请中的候选地址或重新发现。
- 真机连接诊断成功但握手失败时，查看界面的认证错误类型，不要复制完整协议帧或密钥到日志。

## 连接后又断开

- HarmonyOS 进入后台时会按平台边界暂停连接，回到前台后会自动重连。
- Wi-Fi 切换、电脑睡眠或地址变化后，应用会重新发现可信设备并执行断线补同步。
- 被桌面端移除的设备不会继续自动重连，需要生成新邀请重新配对。

## 模拟器说明

模拟器受 VPN/TUN、NAT 和桥接模式影响，可能只有开启 TUN 时才能访问宿主机。这不代表真机也需要 VPN。mDNS、PasteButton、HUKS 和跨设备网络行为最终以真机结果为准。
