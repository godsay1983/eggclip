# EggClip 双端国际化 Roadmap

## 1. 执行规则

本 Roadmap 将 [`双端国际化实现方案.md`](./双端国际化实现方案.md) 拆成八个固定任务。

- 按编号顺序推进；前一任务未完成时不开始后一任务。
- 每轮完成一个完整的 `I18N` 任务，包括实现、自动验证和必要文档同步，然后勾选任务标题和全部子项。
- 子项是同一任务的验收证据，不拆成多个开发轮次。
- 开发期间不新增、删除或拆分 Roadmap 任务。
- 发现阻断缺陷时只写入“阻断记录”，修复后关闭，不增加新任务。
- 不降低配对、密钥、重放、帧上限、剪贴板授权和回环防护。
- 国际化不得修改协议线格式；如发现必须修改，暂停当前任务并先更新总实现方案、schema、双端类型和共享测试向量。
- 不机械翻译用户自定义名称、剪贴板正文、日志样本和测试数据。
- 版本号只在 I18N-01 至 I18N-08 全部完成后提升。

## 2. 当前基线

- 当前统一版本为 `1.0.5`。
- 桌面端 Svelte、Store、Rust command 和原生托盘包含硬编码中文。
- 桌面端部分时间格式固定使用 `zh-CN`。
- HarmonyOS 只有少量 AppScope、Ability 和权限字符串使用资源文件，大部分页面和 Store 仍生成中文。
- 两端设置模型都没有语言选择。
- 部分状态对象保存最终中文 `title/description`，无法响应语言切换。
- 程序生成的默认空间和设备名称可能已经作为中文写入数据库。
- 同步协议不包含语言字段，语言设置也不需要跨设备同步。

## 3. 里程碑

| 里程碑 | 目标 | 完成条件 |
| --- | --- | --- |
| L1 语言基础 | 确定语言、术语、状态码和资源结构 | 两端有可测试的语言基础层 |
| L2 桌面国际化 | Svelte、Rust 错误和托盘完整支持中英文 | 桌面端可即时切换且无混合语言 |
| L3 HarmonyOS 国际化 | 资源、语言偏好、页面和 Store 完成改造 | 手机和平板冷启动后使用正确语言 |
| L4 数据与发布 | 旧名称迁移、门禁、真机回归和文档完成 | 所有完成定义通过 |

## 4. L1：语言基础

### [x] I18N-01 建立术语、语言模型和资源骨架

- [x] 固定 `system`、`zh-CN`、`en-US` 三种设置值和回退规则。
- [x] 落实实现方案中的中英文术语表，统一空间、设备、配对和同步用词。
- [x] 桌面端增加类型化 i18n 目录、语言解析函数和中英文最小词典。
- [x] 桌面 TypeScript/Rust `AppSettings` 增加带默认值的 `languageMode`，旧设置可以读取。
- [x] HarmonyOS 增加 base 英文、`zh_CN` 中文资源骨架和语言偏好 Service。
- [x] HarmonyOS 使用系统应用偏好语言作为事实来源，不在 RDB 重复保存。
- [x] 增加语言解析、默认值和资源骨架单元测试。

完成标准：两端可以读取当前语言、切换最小示例文案，并保持旧设置和现有数据可用。

### [x] I18N-02 把动态状态和错误改为稳定 code

- [x] 定义连接、同步、历史、配对、设备和设置的 `UiMessageCode` 与参数结构。
- [x] 桌面 Store 不再保存最终中文 `title/description`，组件按当前语言渲染 descriptor。
- [x] 定义 Tauri `AppErrorDto` 和稳定错误码，command 边界不再依赖中文字符串匹配。
- [x] HarmonyOS Store 返回状态码和参数，页面通过本地化适配层获取资源。
- [x] HarmonyOS Service 返回领域错误枚举，不直接生成最终 UI 文案。
- [x] 协议错误码和线格式保持不变，不增加自然语言错误正文。
- [x] 覆盖切换语言后已有状态重新渲染、未知 code 安全回退和敏感参数拒绝测试。

完成标准：连接、同步、配对和错误状态与显示语言解耦，切换语言后不保留上一种语言的状态文案。

- [x] 迁移 `aria-label`、`title`、meta description 和图标无障碍文案。
- [x] 删除固定 `zh-CN` 时间格式，统一使用当前语言的 Intl formatter。
- [x] 正确处理在线设备、历史数量和剩余时间的英文单复数。

- [x] 内部诊断保留安全技术上下文，但不把最终中文或英文写入领域状态。

完成标准：桌面面板、Rust 错误和原生托盘始终使用同一语言，语言切换无需重启。

## 9. 完成定义

以下条件必须同时满足：

- I18N-01 至 I18N-08 全部勾选。
- 两端支持跟随系统、简体中文和英语。
- 页面、原生托盘、权限原因、动态状态、错误和无障碍文案完整国际化。
- Store、Service、Rust command 和协议不依赖最终中文或英文文案。
- 日期、时间、数量和单复数使用当前语言规则。
- 程序生成名称可本地化，用户自定义名称不变。
- 语言设置不进入同步协议，不改变安全和剪贴板行为。
- 国际化资源完整性和硬编码扫描进入发布门禁。
- Windows、HarmonyOS 手机和平板真机回归通过。
- README、回归清单和应用商店双语资料与实际行为一致。
