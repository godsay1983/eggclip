# EggClip 双端 UI 精简改造 Roadmap

## 目标

在不改变 v1 核心同步、安全和平台边界的前提下，让桌面端与 HarmonyOS 端从“功能与调试信息并列”收敛为以剪贴板为中心的产品界面。

固定原则：

- 首页只展示当前剪贴板、必要同步状态和最近记录。
- 每张卡片最多一个醒目的主操作；不可用且无解释价值的按钮直接隐藏。
- 普通用户界面不展示 POC、RDB、HMAC、CLIENT_HELLO、AUTH_PROOF 等开发术语。
- 常用设置、设备管理和高级诊断分层，不再堆在同一长滚动区域。
- 重要安全边界保留，但完整说明集中到“隐私与安全”或高级帮助中。
- 不新增业务功能，不修改协议，不突破 HarmonyOS PasteButton 和用户点击复制边界。

## 阶段 1：内容减法与首屏收敛

目标：先删除错误、过时和重复信息，不改变页面导航结构。

### 桌面端

- [x] 删除顶部 Beta、产品边界标签和重复 footer 文案。
- [x] 空剪贴板状态只保留一个有效操作，隐藏无意义的禁用按钮和空闲状态大卡片。
- [x] 设置顶部删除“重新读取”、大块隐私说明和重复保存说明。
- [x] 精简历史空状态、发送状态和技术性文案。

阶段一同时将桌面历史保留从“每个空间分别限制”修正为“所有空间合计限制”，避免界面出现 `21 / 20` 一类超出用户设置上限的状态。

### HarmonyOS

- [x] 首页连接提示压缩为一条状态；未配对时只引导前往设备页。
- [x] 首页隐藏无内容时的“复制到本机”和帧诊断，压缩发送/历史说明。
- [x] 删除配对页已经过时的“仍待后续接入”文案，安全说明改为用户语言。
- [x] 删除设备规则卡，普通页面只保留可信设备、添加设备和折叠诊断。
- [x] 设置页删除 RDB 加载成功提示、重复状态描述和低价值说明。

验收：手机首页首屏能看到剪贴板主操作和最近记录入口；桌面空状态不再出现三个并列按钮；普通页面没有过时开发阶段文案。

## 阶段 2：信息架构重组

### 桌面端

- [x] 设置改为“常规 / 设备 / 高级”三个页签或等价分段导航。
- [x] 可信设备、添加设备和同步空间集中到“设备”。
- [x] 手动 IP、mDNS、网卡、HMAC 和 POC 全部移入“高级”。
- [x] 同步空间卡只显示名称、当前状态和设备摘要，次要操作放入更多菜单。

### HarmonyOS

- [x] 设备页顺序固定为“可信设备 / 添加设备 / 连接问题”。
- [x] 配对流程按“扫描或粘贴 → 核对确认码 → 连接”渐进展示。
- [x] 自动发现、手动 IP、最近地址、VPN/TUN 和帧统计仅在“连接问题”展开后显示。
- [x] 设置页只保留“同步 / 历史 / 外观 / 隐私与安全”，网络诊断移到设备页。
- [x] 评估并移除 HarmonyOS 自动写入系统剪贴板入口，确保与用户点击复制边界一致。

验收：普通用户完成配对不需要理解网络或协议术语；高级诊断默认不占页面高度；桌面设置任一常用操作两次点击内可达。

## 阶段 3：列表、按钮与视觉层级

- [x] 两端历史改为紧凑两行列表，整行作为主操作，详情和删除降级到次级入口。
- [x] 重复解密错误改为短状态，不在每条记录重复整段说明。
- [x] 每张卡片最多一个黄色主按钮；危险操作使用红色文字并二次确认。
- [x] 减少卡片内嵌卡片，页面只保留背景和主卡两级 surface。
- [x] 统一 HarmonyOS 底部导航图标为 24×24 和一致描边，缩短切页动画与底部渐变遮罩。
- [x] 统一桌面 440×680 与 HarmonyOS phone/tablet 的间距、圆角和文本截断规则。

验收：没有两行以上的按钮堆叠；历史列表扫描效率明显提升；页面切换不出现明显双页拼接中间帧。

## 阶段 4：回归与完成标准

- [ ] 桌面浅色/深色、空状态、长文本、历史满载、设置弹层和错误状态截图复核。
- [ ] HarmonyOS 手机/平板、浅色/深色、未配对/在线/断线/待发送状态截图复核。
- [ ] 键盘焦点、屏幕阅读标签、触控热区、长文本和大字体模式检查。
- [ ] 双端完整构建、自动化测试和真机主流程回归。

完成标准：首屏只强调剪贴板；设置与诊断层级清晰；普通用户路径无开发术语；所有既有同步、安全和恢复能力仍可访问。

## 非目标

- 不新增云同步、账号、自动更新、遥测或公网中继。
- 不修改协议、密码算法、数据库 schema 或配对安全模型。
- 不以 UI 精简为由删除排障能力；排障能力移入高级层级。
